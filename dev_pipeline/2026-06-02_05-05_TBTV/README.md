> Documentação de referência para os processos, estrutura e dinâmica de trabalho dos squads.
> Este guia define a forma de trabalho padrão e serve como base para onboarding e alinhamento contínuo.

## Para Quem é Este Guia

- Software Engineers (SEs) de todas as especialidades
- Engineering Manager (EM) e Product Manager (PM)
- Novos membros em processo de onboarding

---

## Sumário

### [01 - Estrutura do Squad](./01-estrutura-do-squad.md)
Composição do time, papéis e responsabilidades, e a filosofia de desenvolvimento generalista.

### [02 - Cerimônias e Cadência](./02-cerimonias-e-cadencia.md)
Ritos ágeis do squad: daily, planning, refinamento técnico, retrospectiva e 1:1s.

### [03 - Fluxo de Trabalho](./03-fluxo-de-trabalho.md)
Board do squad, colunas, critérios de transição entre etapas e responsabilidades de movimentação.

### [04 - Desenvolvimento e Qualidade](./04-desenvolvimento-e-qualidade.md)
Branching strategy, code review, testes, CI/CD, ambientes e padrões de qualidade.

### [05 - Gestão de Demandas](./05-gestao-de-demandas.md)
Tipos de demanda, tratamento de incidentes, bugs e débito técnico.

### [06 - Capacitação e Desenvolvimento](./06-capacitacao-e-desenvolvimento.md)
PDIs, workshops, atuação cross-funcional e ciclo de aprendizado contínuo.
